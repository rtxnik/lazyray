# Support

lazyray (`lzr`) is maintained by a single maintainer. Here is where to go,
depending on what you need.

## Questions and usage help

Ask in [GitHub Discussions (Q&A)](https://github.com/rtxnik/lazyray/discussions/categories/q-a).
Please do not open issues for questions — the issue forms will redirect you.

Before asking, check:

- [README](README.md) — features, installation, quick start
- [TROUBLESHOOTING.md](TROUBLESHOOTING.md) — common failure cases
- [Command reference](docs/reference/README.md) and `lzr <command> --help`
- `lzr doctor` — diagnoses your local environment

## Bugs and feature requests

Open an issue with the matching form:

- [Bug report](https://github.com/rtxnik/lazyray/issues/new?template=bug_report.yml)
- [Feature request](https://github.com/rtxnik/lazyray/issues/new?template=feature_request.yml)

REDACT subscription URLs, server addresses, UUIDs, and passwords from
everything you post — see [SECURITY.md](SECURITY.md).

## Security vulnerabilities

Never open a public issue. Use
[private vulnerability reporting](https://github.com/rtxnik/lazyray/security/advisories/new)
— process and scope in [SECURITY.md](SECURITY.md).

## What is supported

Only the latest released version, fix-forward — see
[SECURITY.md](SECURITY.md#supported-versions) for the support table and
[docs/stability.md](docs/stability.md) for what the version number promises.
